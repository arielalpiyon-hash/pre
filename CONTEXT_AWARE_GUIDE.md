# מדריך מערכת זיהוי הקשר שיחה

## סקירה כללית

מערכת זיהוי הקשר שיחה היא מנגנון חכם שמזהה מידע שהמשתמש מספר על עצמו במהלך השיחה, ומתאים את התשובות בהתאם למצבו האישי. המטרה היא למנוע מצבים שבהם המערכת חוזרת על עצות שכבר לא רלוונטיות למשתמש.

## איך זה עובד?

### שלב 1: זיהוי מידע אוטומטי

המערכת מזהה אוטומטית הצהרות של המשתמש כמו:

#### ✅ יום סיירות
- "עשיתי יום סיירות"
- "עברתי יום סיירות"
- "הייתי ביום סיירות"
- "סיימתי יום סיירות"
- "אחרי יום סיירות"

#### ✅ צו ראשון
- "עשיתי צו ראשון"
- "עברתי צו ראשון"
- "הייתי בצו ראשון"
- "אחרי צו ראשון"

#### ✅ גיבושים
- "עשיתי גיבוש"
- "עברתי גיבוש"
- "הייתי בגיבוש"
- "סיימתי גיבוש"
- "אחרי גיבוש"

#### ✅ זימונים
- "קיבלתי זימון"
- "יש לי זימון"
- "הגיע לי זימון"
- "זימנו אותי"

#### ❌ אי-הצלחה ביום סיירות
- "לא עברתי יום סיירות"
- "נפלתי ביום סיירות"
- "נכשלתי ביום סיירות"
- "לא הצלחתי ביום סיירות"

### שלב 2: שמירת הקשר

כל המידע נשמר בפרופיל המשתמש (`UserProfile`):

```typescript
{

  hasAttendedGibush: true,        // עבר גיבוש

  failedYomSayarot: false,        // לא עבר יום סיירות
  stage: 'post_scout_day',        // שלב נוכחי

}
```

### שלב 3: התאמת תשובות

#### דוגמה 1: משתמש אחרי יום סיירות

**מצב:**
```javascript
userProfile.hasAttendedScoutDay = true
```

**שאלה:**
```
איך מגיעים לשלדג?
```

**תשובה מקורית:**
```
כדי להגיע לשלדג בדרך כלל צריך לעבור יום סיירות ולקבל זימון לגיבוש שלדג.
```

**תשובה מותאמת:**
```
אחרי יום הסיירות, השלב הבא בדרך כלל הוא גיבושים. כדי להגיע לשלדג
בדרך כלל צריך לקבל זימון לגיבוש שלדג.

שלדג היא יחידה מיוחדת של חיל האוויר.

למידע רשמי ועדכני באתר מתגייסים:
https://www.mitgaisim.idf.il/search/?q=שלדג

**השלב הבא:**
אם עברת בהצלחה ביום הסיירות, בדרך כלל מקבלים זימון לגיבושים של היחידות
הרלוונטיות. האם קיבלת כבר זימון לגיבוש?
```

#### דוגמה 2: משתמש שלא עבר יום סיירות

**מצב:**
```javascript
userProfile.failedYomSayarot = true
```

**שאלה:**
```
איך אני יכול להגיע ליחידה טובה?
```

**תשובה מותאמת:**
```
גם אם לא עברת יום סיירות, עדיין יש הרבה אפשרויות מעולות לשירות איכותי.

ישנן אפשרויות רבות לשירות איכותי בצה"ל.

**חשוב לדעת:**
גם ללא יום סיירות, ישנן יחידות מעולות ותפקידים משמעותיים שפתוחים בפניך.
כדאי לבדוק מסלולים נוספים שמתאימים ליכולות שלך.
```

#### דוגמה 3: משתמש אחרי גיבוש

**מצב:**
```javascript
userProfile.hasAttendedGibush = true
```

**שאלה:**
```
מתי אדע אם התקבלתי?
```

**תשובה מותאמת:**
```
אחרי שעברת יום סיירות וגיבוש, בדרך כלל מקבלים תשובה תוך מספר
שבועות לאחר הגיבוש.

**מה הלאה:**
אחרי הגיבוש, בדרך כלל מקבלים הודעה אם התקבלת ליחידה. אם עדיין לא
קיבלת תשובה, כדאי לבדוק עם הגורמים הרלוונטיים.
```

## מבנה טכני

### קבצי מפתח

#### 1. `/src/types/conversation.ts`
מגדיר את מבנה הנתונים:

```typescript
export interface UserProfile {
  hasAttendedScoutDay?: boolean;
  hasAttendedGibush?: boolean;
  hasAttendedTsavRishon?: boolean;
  hasReceivedZimun?: boolean;
  failedYomSayarot?: boolean;
  profileLevel?: string;
  interestedUnits?: string[];
  stage?: 'pre_draft' | 'post_scout_day' | 'post_gibush' |
          'waiting_for_zimun' | 'post_draft' | 'in_service';
  strengths?: string[];
  concerns?: string[];
}

export const CONTEXT_KEYWORDS = {
  hasAttendedScoutDay: [...],
  hasAttendedTsavRishon: [...],
  hasAttendedGibush: [...],
  hasReceivedZimun: [...],
  failedYomSayarot: [...]
};
```

#### 2. `/src/utils/conversationContext.ts`
פונקציות לעיבוד הקשר:

```typescript
// זיהוי הקשר מהודעת משתמש
export function extractUserContext(
  question: string,
  context: ConversationContext
): Partial<UserProfile>

// יצירת קידומת הקשרית
export function getContextualPrefix(
  profile: UserProfile,
  questionTopic: string
): string

// התאמת תשובה לפי הקשר
export function adaptAnswerToContext(
  answer: string,
  profile: UserProfile,
  question: string
): string

// בדיקה אם צריך לשאול שאלת המשך
export function shouldAskFollowUpQuestion(
  profile: UserProfile
): { shouldAsk: boolean; question?: string }
```

#### 3. `/src/utils/agentSearch.ts`
שילוב במערכת החיפוש:

```typescript
export async function searchWithAgent(
  question: string,
  persona: 'candidate' | 'parent',
  userProfile?: UserProfile // פרמטר חדש
): Promise<string>
```

#### 4. `/src/components/AskQuestionModal.tsx`
רכיב השיחה שמשתמש במנגנון:

```typescript
const userContextUpdates = extractUserContext(questionToAsk, conversationContext);
const updatedContext = updateUserProfile(conversationContext, userContextUpdates);

const rawAnswer = await searchWithAgent(
  questionToAsk,
  userType,
  updatedContext.userProfile
);
```

## שלבי העיבוד

```
1. משתמש שולח הודעה
   ↓
2. extractUserContext() - זיהוי הקשר בהודעה
   ↓
3. updateUserProfile() - עדכון פרופיל המשתמש
   ↓
4. searchWithAgent() - חיפוש תשובה + העברת הפרופיל
   ↓
5. enrichAnswerWithUnitInfo() - העשרה עם מידע יחידות
   ↓
6. adaptAnswerToContext() - התאמה לפי הקשר
   ↓
7. הצגת התשובה למשתמש
```

## תרחישים נתמכים

### תרחיש 1: התקדמות טבעית
```
משתמש: "עשיתי יום סיירות"
→ המערכת: מזהה → hasAttendedScoutDay = true

משתמש: "איך מגיעים לשייטת?"
→ המערכת: מתאימה תשובה לשלב הבא (גיבושים)

משתמש: "קיבלתי זימון לגיבוש"
→ המערכת: מזהה → hasReceivedZimun = true

משתמש: "איך להתכונן?"
→ המערכת: מתאימה תשובה להכנה לגיבוש
```

### תרחיש 2: אי-הצלחה ביום סיירות
```
משתמש: "לא עברתי יום סיירות"
→ המערכת: מזהה → failedYomSayarot = true

משתמש: "איך אני יכול להגיע ליחידה טובה?"
→ המערכת: מציעה מסלולים חלופיים, לא מדברת על יום סיירות
```

### תרחיש 3: משתמש אחרי גיבוש
```
משתמש: "עשיתי גיבוש לשייטת"
→ המערכת: מזהה → hasAttendedGibush = true, interestedUnits = ['שייטת']

משתמש: "מתי אדע אם התקבלתי?"
→ המערכת: מתאימה תשובה לשלב אחרי גיבוש
```

## שילוב עם העשרת יחידות

המערכת משלבת את שני המנגנונים:

1. **זיהוי הקשר** - מזהה את מצב המשתמש
2. **העשרת יחידות** - מוסיף מידע ספציפי על יחידות
3. **התאמת הקשר** - מתאים את התשובה למצב המשתמש

### דוגמה מלאה:

**משתמש:** "עשיתי יום סיירות"
**משתמש:** "איך מגיעים לשלדג?"

**תשובה:**
```
אחרי יום הסיירות, השלב הבא בדרך כלל הוא גיבושים. כדי להגיע לשלדג
בדרך כלל צריך לקבל זימון לגיבוש שלדג.

שלדג היא יחידה מיוחדת של חיל האוויר.
               ↑ מהעשרת יחידות

למידע רשמי ועדכני באתר מתגייסים:
https://www.mitgaisim.idf.il/search/?q=שלדג
               ↑ מהעשרת יחידות

**השלב הבא:**
אם עברת בהצלחה ביום הסיירות, בדרך כלל מקבלים זימון לגיבושים של
היחידות הרלוונטיות. האם קיבלת כבר זימון לגיבוש?
               ↑ מהתאמת הקשר
```

## יתרונות המערכת

1. **שיחה טבעית** - המערכת "זוכרת" מה המשתמש אמר
2. **תשובות רלוונטיות** - אין עצות מיותרות או חוזרות
3. **התקדמות הגיונית** - המערכת מכוונת למשתמש לשלב הבא
4. **חוויה אישית** - כל משתמש מקבל תשובות מותאמות למצבו
5. **יעילות** - חסכון בזמן ובבלבול של המשתמש

## בדיקת המערכת

### תרחישי בדיקה מומלצים:

#### בדיקה 1: משתמש אחרי יום סיירות
```
1. פתח את האפליקציה
2. בשיחה, כתוב: "עשיתי יום סיירות"
3. שאל: "איך מגיעים לשלדג?"
4. בדוק שהתשובה לא מדברת על "לעבור יום סיירות"
5. בדוק שהתשובה מתמקדת בגיבושים
6. בדוק שיש קישור לאתר מתגייסים
```

#### בדיקה 2: משתמש שלא עבר יום סיירות
```
1. פתח שיחה חדשה
2. כתוב: "לא עברתי יום סיירות"
3. שאל: "איך אני יכול לשרת ביחידה טובה?"
4. בדוק שהתשובה מעודדת ומציעה אפשרויות חלופיות
5. בדוק שאין התייחסות ליום סיירות כדרישה
```

#### בדיקה 3: שרשרת שיחה מלאה
```
1. "עשיתי צו ראשון" → המערכת מזהה
2. "עשיתי יום סיירות" → המערכת מזהה ומעדכנת שלב
3. "איך מגיעים למטכל?" → תשובה מותאמת לאחר יום סיירות
4. "קיבלתי זימון לגיבוש" → המערכת מזהה
5. "איך להתכונן?" → תשובה ספציפית להכנה לגיבוש
```

## סימנים לבעיה

❌ המערכת נותנת עצה "לעבור יום סיירות" למשתמש שכבר עבר
❌ התשובה לא מתייחסת למידע שהמשתמש סיפר
❌ אין התאמה לשלב הנוכחי של המשתמש
❌ המערכת לא שואלת שאלות המשך רלוונטיות

## הוספת הקשרים חדשים

להוספת זיהוי הקשר חדש:

### 1. הוסף ל-`UserProfile`:
```typescript
export interface UserProfile {
  // ...קיים
  newContext?: boolean;  // הוסף שדה חדש
}
```

### 2. הוסף מילות מפתח ל-`CONTEXT_KEYWORDS`:
```typescript
export const CONTEXT_KEYWORDS = {
  // ...קיים
  newContext: [
    'מילת מפתח 1',
    'מילת מפתח 2',
    'מילת מפתח 3'
  ]
};
```

### 3. הוסף זיהוי ב-`extractUserContext()`:
```typescript
if (CONTEXT_KEYWORDS.newContext.some(keyword => lowerQuestion.includes(keyword))) {
  updates.newContext = true;
}
```

### 4. הוסף לוגיקה ב-`adaptAnswerToContext()`:
```typescript
if (profile.newContext) {
  return `${contextPrefix}${answer}\n\n**הערה מותאמת:**\n...`;
}
```
