# מדריך למנגנון העשרת תשובות

## סקירה כללית

מנגנון העשרת תשובות הוא מערכת אוטומטית שמזהה יחידות צבאיות, תפקידים ומסלולי שירות בתשובות שהמערכת מחזירה למשתמשים. כאשר מזוהה יחידה, המערכת מוסיפה:

1. **מידע קצר** על סוג היחידה
2. **קישור** לאתר מתגייסים לקבלת מידע רשמי ועדכני

## איך זה עובד?

## עקרונות ומגבלות

### עקרונות חשובים
1. **קישור יחיד** - המערכת יוצרת רק קישור אחד לכל תשובה
2. **אי-שינוי התוכן** - התשובה המקורית לא משתנה, רק מתווסף מידע
3. **הימנעות מכפילויות** - אם כבר קיים קישור לאתר מתגייסים, לא מוסיף נוסף
4. **זיהוי חכם** - המערכת מזהה וריאציות שונות של שמות יחידות (למשל: מטכ״ל, מטכ"ל, מטכל)

### תרחישים שבהם לא מתווסף מידע
- כאשר לא זוהתה יחידה ספציפית
- כאשר כבר קיים קישור לאתר מתגייסים בתשובה
- כאשר כבר קיים תיאור היחידה בתשובה

## מיקום הקוד

הפונקציונליות ממוקמת בקבצים הבאים:

### `/src/utils/unitEnricher.ts`
קובץ ייעודי למנגנון העשרת תשובות. מכיל:
- `enrichAnswerWithUnitInfo()` - פונקציה מרכזית להעשרת תשובות
- `getUnitInfoForText()` - פונקציה להחזרת מידע על יחידה
- `UNITS_DATABASE` - מאגר היחידות ותיאוריהן

### `/src/utils/agentSearch.ts`
משולב במערכת החיפוש הראשית:
```typescript
import { enrichAnswerWithUnitInfo } from './unitEnricher';

// בסוף תהליך החיפוש:
return enrichAnswerWithUnitInfo(finalAnswer, question);
```

## הוספת יחידות חדשות

להוספת יחידה חדשה, יש להוסיף רשומה ל-`UNITS_DATABASE` בקובץ `unitEnricher.ts`:

```typescript
{
  names: ['שם היחידה', 'וריאציה 1', 'וריאציה 2'],
  description: 'תיאור קצר של היחידה',
  searchTerm: 'מונח_לחיפוש' // ללא רווחים, עם + במקום רווחים
}
```

## יתרונות המערכת

1. **שיפור אמינות** - הפניה לאתר מתגייסים הרשמי
2. **מידע עדכני** - המשתמשים מקבלים קישור למידע הרשמי ביותר
3. **שקיפות** - ברור למשתמש מהו מקור המידע הרשמי
4. **עקביות** - כל תשובה שמזכירה יחידה מקבלת אותו סוג של העשרה
5. **חוויית משתמש משופרת** - מידע נוסף מועיל ללא עומס יתר
