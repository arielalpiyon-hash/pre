# מדריך מערכת הקשר שיחה פשוטה

## עקרונות בסיסיים

המערכת משתמשת ב-**3 משתני הקשר פשוטים** כדי לזכור את מצב המשתמש ולהתאים תשובות:

```typescript
did_tzav_rishon: null | true | false
did_yom_sayarot: null | true | false
did_get_gibush: null | true | false
```

### ערכים אפשריים:
- `null` - לא יודעים עדיין
- `true` - המשתמש עשה את השלב
- `false` - המשתמש עדיין לא עשה

---

## זיהוי אוטומטי

### צו ראשון

**אם המשתמש אומר:**
```
✅ "עשיתי צו ראשון"
✅ "עברתי צו ראשון"
→ did_tzav_rishon = true

❌ "עוד לא עשיתי צו ראשון"
❌ "לא עשיתי צו ראשון"
→ did_tzav_rishon = false
```

### יום סיירות

**אם המשתמש אומר:**
```
✅ "עשיתי יום סיירות"
✅ "עברתי יום סיירות"
→ did_yom_sayarot = true

❌ "לא עשיתי יום סיירות"
❌ "נפלתי ביום סיירות"
→ did_yom_sayarot = false
```

### גיבוש

**אם המשתמש אומר:**
```
✅ "קיבלתי גיבוש"
✅ "יש לי גיבוש"
→ did_get_gibush = true

❌ "לא קיבלתי גיבוש"
❌ "אין לי גיבוש"
→ did_get_gibush = false
```

---

## חוקי התאמת תשובות

### חוק 1: אם עשה יום סיירות
**אסור:** "צריך לעבור יום סיירות"
**במקום:** "אם כבר עשית יום סיירות, השלב הבא תלוי בתוצאות..."

### חוק 2: אם קיבל גיבוש
**אסור:** "השלב הבא הוא לקבל גיבוש"
**במקום:** "אם כבר קיבלת גיבוש, הפוקוס על הכנה..."

### חוק 3: אם לא עשה צו ראשון
**כדאי להוסיף:** "השלבים הראשונים מתחילים בצו ראשון."

---

## שאלות המשך

המערכת שואלת **רק שאלה אחת** כשחסר מידע חשוב.

### סדר עדיפויות:
1. "כבר עשית צו ראשון?"
2. "כבר עשית יום סיירות?"
3. "כבר קיבלת גיבוש?"

---

## דוגמאות

### משתמש חדש:
```
משתמש: "איך מגיעים לשלדג?"
תשובה: [תשובה רגילה]

כדי לכוון אותך נכון:
כבר עשית צו ראשון?
```

### משתמש שעשה יום סיירות:
```
משתמש: "עשיתי יום סיירות"
משתמש: "איך מגיעים לשלדג?"

תשובה:
אם כבר עשית יום סיירות, השלב הבא תלוי בתוצאות שקיבלת.
[תשובה ללא עצה "לעבור יום סיירות"]

כדי לכוון אותך נכון:
כבר קיבלת גיבוש?
```

### משתמש עם גיבוש:
```
משתמש: "קיבלתי גיבוש"
משתמש: "איך מגיעים לשלדג?"

תשובה:
אם כבר קיבלת גיבוש, הפוקוס על הכנה לגיבוש.
[מידע על הכנה לגיבוש]
```

המערכת מספקת חוויית שיחה אישית ומותאמת!
